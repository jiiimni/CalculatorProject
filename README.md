# CH2 Java 계산기 과제

내일배움캠프 Spring 3기 CH2 계산기 과제 
자바 기본 문법부터 객체지향 개념, 컬렉션, 제네릭, 람다 & 스트림까지  
단계적으로 학습하고 적용하는 것을 목표로 진행했습니다.

---

## 과제 목표

- Java 기본 문법 이해 및 활용
- 객체지향 프로그래밍(OOP) 개념 적용
- 컬렉션을 활용한 데이터 관리
- Enum, 제네릭, 람다 & 스트림의 실제 사용 경험
- 단계별 요구사항을 충족하는 코드 작성

---

## STEP 1. 클래스 없이 기본 계산기 구현

**목표**
- 클래스 없이 Java 기본 문법만을 사용해 계산기 구현

**주요 내용**
- 사용자로부터 숫자 2개와 사칙연산 기호 입력
- `switch` 문을 활용한 사칙연산 처리
- `while` 반복문을 사용해 `exit` 입력 전까지 계속 계산
- 0으로 나누기, 잘못된 연산자 입력에 대한 예외 처리

**특징**
- 모든 로직을 `App` 클래스의 `main` 메서드 내부에서 구현
- 콘솔 기반 사용자 인터페이스

---

## STEP 2. 클래스를 적용한 계산기 구현

**목표**
- 객체지향 개념을 적용해 계산 로직을 클래스 단위로 분리

**주요 내용**
- `Calculator` 클래스 생성
  - 사칙연산 수행 메서드 구현
  - 연산 결과를 저장하는 `List` 컬렉션 필드 추가
- `App` 클래스에서 `Calculator`를 활용하도록 수정
- 연산 결과 컬렉션에 대한 캡슐화 적용
  - `private` 필드
  - `getter / setter`를 통한 간접 접근
- 저장된 연산 결과 중 가장 먼저 저장된 값 삭제(removeResult) 기능 구현

**특징**
- STEP1과 동일한 사용자 입력/출력 흐름 유지
- 내부 구조만 객체지향적으로 개선

---

## STEP 3. Enum, 제네릭, 람다 & 스트림 적용

**목표**
- Java의 고급 문법을 계산기에 적용해 구조 확장

**주요 내용**
- **Enum**
  - 사칙연산 타입을 `OperatorType` enum으로 관리
- **제네릭**
  - `ArithmeticCalculator<T extends Number>` 형태로 계산기 확장
  - 정수뿐 아니라 실수(`double`) 연산도 가능하도록 구현
- **람다 & 스트림**
  - 저장된 연산 결과 중
    특정 값보다 큰 결과만 조회하는 기능 구현
  - Stream API를 활용해 조건 필터링 처리

**특징**
- 기존 구조를 유지한 채 기능 확장
- 계산 책임과 데이터 관리 책임을 명확히 분리
